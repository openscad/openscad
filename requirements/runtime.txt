# Runtime Python dependencies bundled / declared by PythonSCAD.
#
# Only place packages here that are needed at runtime by the embedded
# CPython interpreter (i.e. that the user's Python code can `import`).
# Build-time-only tools belong elsewhere.
#
# Pinning policy: pin to the major version range. Patch and minor
# upgrades are picked up automatically by Dependabot (configured in
# .github/dependabot.yaml), which re-runs CI to catch any regressions.
#
# This file is consumed by:
#   * pyproject.toml [project.dependencies]     (published pip wheel,
#                                                kept in sync manually)
#   * scripts/bundle-runtime-python.sh          (canonical bundle helper,
#                                                called by every binary pipeline)
#   * .github/workflows/build-appimage.yml      (via scripts/create_appimage.sh)
#   * .github/workflows/build-macos-release.yml (per-arch + universal .app)
#   * .github/workflows/build-windows-native.yml (CPack staging dir)
#   * .github/dependabot.yaml                   (auto-bump watcher)
#
# Distribution-native packages (.deb, .rpm) deliberately do NOT consume
# this file; they declare a `Recommends:` on `python3-ipython` instead
# so the distro packagers stay in control of the dependency.

# IPython provides the rich interactive shell launched by
# `pythonscad --ipython`. The basic embedded REPL launched by
# `pythonscad --repl` works without any optional packages.
#
# IPython 9.x added a hard runtime dependency on `psutil>=7`. On
# Linux/macOS/PyPI, pip resolves psutil from prebuilt wheels. On
# Windows native (MSYS2 UCRT64), PyPI has no psutil wheel for the
# `mingw_x86_64_ucrt_gnu-cpython-3xx` ABI and source builds fail
# against MSYS2's `winternl.h`. scripts/bundle-runtime-python.sh
# therefore vendors psutil from the MSYS2 pacman package
# (`pacboy -S --noconfirm python-psutil:p`) on that platform; see
# scripts/bundle-ipython-msys2.py.
ipython>=9,<10
